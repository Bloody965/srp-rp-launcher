1. Открой лаунчер
2. Нажми "Нет аккаунта? Регистрация"

### Rate Limiting:
- Регистрация: 3 попытки в час с одного IP
- Вход: 5 попыток в 15 минут с одного IP

## 📊 База данных:

SQLite база: `D:\лаунчер\src\ApocalypseLauncher.API\apocalypse_launcher.db`

**Таблицы:**
- `Users` - пользователи (id, username, email, passwordHash, minecraftUUID, isActive, isBanned, isWhitelisted)
- `LoginSessions` - активные сессии (id, userId, token, ipAddress, expiresAt)
- `AuditLogs` - логи действий (id, userId, action, details, ipAddress, createdAt)
- `ModpackVersions` - версии модпаков (id, version, downloadUrl, sha256Hash, fileSizeBytes, changelog, isActive)

## 🔍 Просмотр логов:

Все действия логируются в таблицу `AuditLogs`:
- REGISTER - регистрация
- REGISTER_SQL_INJECTION_ATTEMPT - попытка SQL инъекции
- REGISTER_RATE_LIMITED - превышен лимит регистраций
- LOGIN_SUCCESS - успешный вход
- LOGIN_FAILED - неудачная попытка входа
- LOGIN_BANNED - попытка входа забаненного пользователя
- LOGIN_RATE_LIMITED - превышен лимит попыток входа
- MODPACK_VERSION_CHECK - проверка версии модпака
- MODPACK_DOWNLOAD - скачивание модпака
- MODPACK_VERIFY - проверка хеша модпака

## 🚨 Troubleshooting:

### API не запускается:
- Проверь что порт 5000 свободен
- Проверь что .NET 8.0 установлен

### Лаунчер не подключается к API:
- Убедись что API запущен (http://localhost:5000/api/health должен отвечать)
- Проверь firewall

### Ошибка "Токен недействителен":
- Токен истек (24 часа)
- Войди заново

### Ошибка "Слишком много попыток":
- Подожди 15 минут
- Или перезапусти API сервер (сбросит rate limit)

## 📦 Production:

Для продакшена нужно:
1. Залить API на сервер (Railway/Render/VPS)
2. Получить домен с SSL сертификатом
3. Изменить URL в лаунчере с `http://localhost:5000` на `https://your-domain.com`
4. Настроить CORS для конкретного домена
5. Включить whitelist систему
6. Обновить JWT пакеты (есть уязвимость в 7.0.3)
