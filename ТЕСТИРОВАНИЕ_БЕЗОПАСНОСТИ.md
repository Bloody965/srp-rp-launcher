# Тестирование системы безопасности

## 1. Тест защиты от SQL инъекций

### Попытка SQL инъекции при регистрации:

**PowerShell:**
```powershell
$body = @{
    username = "admin' OR '1'='1"
    email = "test@test.com"
    password = "Test123456"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:5000/api/auth/register" -Method POST -Body $body -ContentType "application/json"
```

**Ожидаемый результат:** Ошибка "Обнаружена попытка SQL инъекции"

### Другие SQL инъекции:
```powershell
# Попытка 1: DROP TABLE
$body = @{
    username = "test'; DROP TABLE Users; --"
    email = "test@test.com"
    password = "Test123456"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:5000/api/auth/register" -Method POST -Body $body -ContentType "application/json"

# Попытка 2: UNION SELECT
$body = @{
    username = "test' UNION SELECT * FROM Users--"
    email = "test@test.com"
    password = "Test123456"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:5000/api/auth/register" -Method POST -Body $body -ContentType "application/json"
```

## 2. Тест Rate Limiting

### Тест защиты от брутфорса (5 попыток в 15 минут):

**PowerShell:**
```powershell
# Попытка войти 6 раз подряд с неверным паролем
for ($i = 1; $i -le 6; $i++) {
    Write-Host "Попытка $i..."
    
    $body = @{
        username = "testuser"
        password = "wrongpassword$i"
    } | ConvertTo-Json
    
    try {
        $response = Invoke-RestMethod -Uri "http://localhost:5000/api/auth/login" -Method POST -Body $body -ContentType "application/json"
        Write-Host "Ответ: $($response.message)"
    } catch {
        Write-Host "Ошибка: $($_.Exception.Response.StatusCode)"
    }
    
    Start-Sleep -Seconds 1
}
```

**Ожидаемый результат:** 
- Попытки 1-5: "Неверное имя пользователя или пароль"
- Попытка 6: "Слишком много попыток входа. Попробуйте через 15 минут."

## 6. Тест успешной регистрации и входа

### Успешная регистрация:

**PowerShell:**
```powershell
$body = @{
    username = "TestPlayer123"
    email = "player@example.com"
    password = "SecurePass123"
} | ConvertTo-Json

$response = Invoke-RestMethod -Uri "http://localhost:5000/api/auth/register" -Method POST -Body $body -ContentType "application/json"

Write-Host "Успех: $($response.success)"
Write-Host "Токен: $($response.token)"
Write-Host "Username: $($response.user.username)"
Write-Host "UUID: $($response.user.minecraftUUID)"
```

### Успешный вход:

**PowerShell:**
```powershell
$body = @{
    username = "TestPlayer123"
    password = "SecurePass123"
} | ConvertTo-Json

$response = Invoke-RestMethod -Uri "http://localhost:5000/api/auth/login" -Method POST -Body $body -ContentType "application/json"

Write-Host "Успех: $($response.success)"
Write-Host "Токен: $($response.token)"

# Сохраняем токен для следующих тестов
$token = $response.token
```

### Использование токена:

**PowerShell:**
```powershell
$headers = @{
    "Authorization" = "Bearer $token"
}

$response = Invoke-RestMethod -Uri "http://localhost:5000/api/auth/verify" -Method POST -Headers $headers

Write-Host "Токен валиден: $($response.success)"
```

## 9. Автоматический тест всех защит

**Сохрани как test_security.ps1:**

```powershell
Write-Host "=== ТЕСТИРОВАНИЕ СИСТЕМЫ БЕЗОПАСНОСТИ ===" -ForegroundColor Cyan

$baseUrl = "http://localhost:5000"
$passed = 0
$failed = 0

function Test-Security {
    param($name, $scriptBlock)
    
    Write-Host "`n[$name]" -ForegroundColor Yellow
    try {
        & $scriptBlock
        Write-Host "✓ PASSED" -ForegroundColor Green
        $script:passed++
    } catch {
        Write-Host "✗ FAILED: $($_.Exception.Message)" -ForegroundColor Red
        $script:failed++
    }
}

# Тест 1: SQL Injection
Test-Security "SQL Injection Protection" {
    $body = @{
        username = "admin' OR '1'='1"
        email = "test@test.com"
        password = "Test123456"
    } | ConvertTo-Json
    
    $response = Invoke-RestMethod -Uri "$baseUrl/api/auth/register" -Method POST -Body $body -ContentType "application/json" -ErrorAction Stop
    
    if ($response.message -like "*SQL*") {
        Write-Host "SQL инъекция заблокирована"
    } else {
        throw "SQL инъекция не заблокирована!"
    }
}

# Тест 2: Слабый пароль
Test-Security "Weak Password Rejection" {
    $body = @{
        username = "testuser"
        email = "test@test.com"
        password = "weak"
    } | ConvertTo-Json
    
    try {
        $response = Invoke-RestMethod -Uri "$baseUrl/api/auth/register" -Method POST -Body $body -ContentType "application/json" -ErrorAction Stop
        throw "Слабый пароль принят!"
    } catch {
        if ($_.Exception.Message -like "*пароль*") {
            Write-Host "Слабый пароль отклонен"
        } else {
            throw
        }
    }
}

# Тест 3: Успешная регистрация
Test-Security "Successful Registration" {
    $username = "TestUser$(Get-Random -Maximum 9999)"
    $body = @{
        username = $username
        email = "$username@test.com"
        password = "SecurePass123"
    } | ConvertTo-Json
    
    $response = Invoke-RestMethod -Uri "$baseUrl/api/auth/register" -Method POST -Body $body -ContentType "application/json" -ErrorAction Stop
    
    if ($response.success -and $response.token) {
        Write-Host "Регистрация успешна, токен получен"
        $script:testToken = $response.token
        $script:testUsername = $username
    } else {
        throw "Регистрация не удалась"
    }
}

# Тест 4: Проверка токена
Test-Security "Token Validation" {
    $headers = @{
        "Authorization" = "Bearer $script:testToken"
    }
    
    $response = Invoke-RestMethod -Uri "$baseUrl/api/auth/verify" -Method POST -Headers $headers -ErrorAction Stop
    
    if ($response.success) {
        Write-Host "Токен валиден"
    } else {
        throw "Токен невалиден"
    }
}

# Тест 5: Доступ без токена
Test-Security "Unauthorized Access Block" {
    try {
        $response = Invoke-RestMethod -Uri "$baseUrl/api/modpack/version" -Method GET -ErrorAction Stop
        throw "Доступ без токена разрешен!"
    } catch {
        if ($_.Exception.Response.StatusCode -eq 401) {
            Write-Host "Доступ без токена заблокирован"
        } else {
            throw
        }
    }
}

Write-Host "`n=== РЕЗУЛЬТАТЫ ===" -ForegroundColor Cyan
Write-Host "Пройдено: $passed" -ForegroundColor Green
Write-Host "Провалено: $failed" -ForegroundColor Red

if ($failed -eq 0) {
    Write-Host "`n✓ ВСЕ ТЕСТЫ ПРОЙДЕНЫ!" -ForegroundColor Green
} else {
    Write-Host "`n✗ ЕСТЬ ПРОБЛЕМЫ С БЕЗОПАСНОСТЬЮ!" -ForegroundColor Red
}
```

**Запуск:**
```powershell
.\test_security.ps1
```

## 10. Проверка хешей паролей

**Открой базу данных и проверь:**

```sql
SELECT Username, PasswordHash FROM Users LIMIT 5;
```

**Хеш должен выглядеть так:**
```
$2a$12$abcdefghijklmnopqrstuvwxyz1234567890ABCDEFGHIJKLMNOP
```

- Начинается с `$2a$` или `$2b$` (Bcrypt)
- Содержит `$12$` (12 раундов)
- Длина ~60 символов
- **НЕ** должен быть MD5 (32 символа hex)
